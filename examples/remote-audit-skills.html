LLM Security Review

HIGH
  • The skill instructs agents to run arbitrary shell commands (git clone, npx, shell scripts) from an external repository without any integrity verification, hash checking, or signature validation.
    Fix: Add checksum verification for downloaded scripts and pin to specific commit hashes rather than mutable branch heads.
  • The install scripts (codex-install.sh, gemini-install.sh) are executed directly after cloning but their contents are never shown or described, making their actual behavior opaque to the user and the reviewing agent.
    Fix: Document exactly what each install script does, or replace opaque shell scripts with explicit inline commands.
  • Installing skills to ~/.claude/skills/ allows third-party code to inject persistent instructions into all future Claude Code sessions without per-session user consent.
    Fix: Warn users that installing to global skill directories grants persistent instruction-injection capability to the skill author.
MEDIUM
  • The skill is named 'Installation' but functions as a promotional document for a third-party skill marketplace, directing users to install 205+ unaudited skills from an external GitHub repository.
    Fix: Rename to clearly indicate this is a third-party plugin catalog, and add warnings that installed skills have not been security-reviewed.
  • The skill references Claude Code features that do not exist (/plugin marketplace, /plugin install, /plugin update, /plugin remove), which would confuse both agents and users.
    Fix: Remove or clearly label fictitious commands; only reference real, documented Claude Code functionality.
  • The universal installer claims to write to sensitive agent configuration directories across 37+ agents simultaneously, but the scope of filesystem changes is never enumerated.
    Fix: Require explicit per-agent opt-in and show the full list of paths that will be modified before execution.
  • The npx command executes a remote npm package (agent-skills-cli) without version pinning, allowing a supply-chain attack if the package is compromised.
    Fix: Pin to a specific version (e.g., npx agent-skills-cli@1.2.3) and verify the package publisher's identity.
LOW
  • The document is truncated mid-sentence at the end, suggesting incomplete content that an agent might attempt to fetch or complete unpredictably.
    Fix: Complete the truncated content or add a clear end-of-document marker.
